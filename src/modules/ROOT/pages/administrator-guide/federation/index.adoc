[#federation]
= {PRODUCT_NAME} {APP_NAME_FEDERATION}

[NOTE]
====
{PRODUCT_NAME} {APP_NAME_FEDERATION} is disabled by default.
Set the `FEDERATION_ENABLED` `.env` variable to `true` if you wish to enable it.
====

{PRODUCT_NAME} {APP_NAME_MESSAGING} aims to connect different Corteza instances into a federated network, where they can freely exchange information.

This document will help you through the process of *setting up your federated network* and *configuring access control* for data sharing.

.Quick access:
* <<node-pairing,Adding new federated nodes>>,
* <<expose-module,Configuring data sharing>>,
* <<module-mapping,Consuming shared data>>.
